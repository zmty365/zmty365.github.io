容器概述

容器是什么

容器是一种受隔离（namespace）且受约束（cgroups）的进程组，也是隔离且受约束的环境，可在其中运行一个或多个进程。

容器技术底层

容器的实现依赖于 Linux 内核提供的两个核心功能：Namespace 和 Cgroup。这些技术共同支撑了容器的隔离性和资源管理机制，此外，UnionFS（联合文件系统） 是实现镜像轻量化的关键。

1. Namespace：实现“视图隔离”

作用是为容器提供独立的“系统视图”，让容器内进程误以为自己在独立的操作系统中运行，实现资源隔离。常见的 Namespace 类型及功能如下：

命名空间类型 作用（隔离的资源） 典型场景
PID（进程 ID） 隔离进程 ID 编号空间。每个命名空间内的进程有独立的 PID 序列（如各自的 PID=1），且看不到其他命名空间的进程。 容器内的进程 PID 与主机隔离（如容器内的 init 进程为 PID=1）。
Mount（挂载） 隔离文件系统挂载点。不同命名空间的进程看到的挂载点（文件系统结构）可以不同。 容器拥有独立的根文件系统（如 / 目录与主机不同）。
UTS（主机名/域名） 隔离主机名和域名。每个命名空间可以设置独立的 hostname。 容器可以有自己的主机名（如 docker run --hostname mycontainer）。
IPC（进程间通信） 隔离 System V IPC 和 POSIX 消息队列。不同命名空间的进程无法通过 IPC 直接通信。 避免容器内进程与主机或其他容器的 IPC 冲突。
Network（网络） 隔离网络资源（网卡、IP、端口、路由表、防火墙规则等）。每个命名空间有独立的网络栈。 容器有独立的虚拟网卡和 IP（如 Docker 的桥接网络）。
User（用户/组 ID） 隔离用户 ID（UID）和组 ID（GID）。命名空间内的 UID/GID 可以与外部不同（如容器内的 root 对应主机的普通用户）。 容器内的 root 用户在主机上无特权，提高安全性。
Cgroup（控制组） 隔离 cgroup 视图。进程只能看到自己所在的 cgroup 层级，无法访问其他命名空间的 cgroup 资源。 限制容器对系统资源（CPU、内存）的使用范围。
Time（时间） 隔离系统时间（包括时钟和计时器）。可以在命名空间内修改时间而不影响主机。 测试依赖时间的程序（如日志、证书过期）时，避免影响主机时间。

2. Cgroup：实现“资源约束”

全称为 Control Groups，作用是限制、统计和隔离容器对 CPU、内存、磁盘 I/O 等资源的使用，避免单个容器过度占用主机资源。常见的 Cgroup 子系统及功能如下：

  • CPU 子系统：限制容器可使用的 CPU 时间片（如限制使用 1 个 CPU 核心、CPU 使用率不超过 50%）。
  • Memory 子系统：限制容器最大可用内存（如限制为 1GB），内存超限时可触发容器 OOM（Out Of Memory）杀死进程。
  • BlkIO 子系统：限制容器对磁盘的读写速率（如限制写速率不超过 100MB/s），避免磁盘 I/O 被单个容器占满。
  • PID 子系统：限制容器内最大进程数（如限制最多运行 100 个进程），防止容器内进程泄露耗尽主机资源。

3. UnionFS：实现“镜像分层”

联合文件系统（如 AUFS、Overlay2）是容器镜像轻量化的核心技术，它支持将多个只读的“镜像层”叠加为一个可写的“容器层”，实现分层存储：

  • 镜像层（只读）：镜像由多个分层组成（如基础 OS 层、依赖库层、应用代码层），分层可被多个容器共享（如 10 个 Nginx 容器可共享同一个 Nginx 基础镜像层），减少磁盘占用。
  • 容器层（可写）：容器运行时会在镜像层之上添加一个可写层，容器内的文件修改（如创建、删除文件）仅发生在可写层，镜像层保持不变——这也是“容器删除后修改丢失，镜像可重复创建容器”的原因。

在下一节，我们将了解容器的底层原理之一——Namespace，通过动手实践逐步搭建一个隔离的环境。
